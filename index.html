    • Easy Question

  • 1. What is the main method?
    Answer

    The main method is the program’s entry point — the place where the Java Virtual Machine starts running your program. Think of it as the program’s front door. The JVM looks for a method with the signature public static void main(String[] args) and starts from there.

  • 2. What does the signature public static void main(String[] args) mean (briefly)?
    Answer

    • public — the JVM must be able to access it.
    • static — it belongs to the class itself (no object needed).
    • void — it doesn’t return a value to the JVM.
    • main — the method name JVM expects.
    • String[] args — an array that holds command-line arguments.

  • 3. Why does main have to be static?
    Answer

    Because the JVM calls main before creating any object of the class. Marking it static lets the JVM call it using the class name (no instance required) — like turning on a machine from its power switch rather than needing a worker already inside the machine.

  • 4. Why is the return type void? How do I signal a status code then?
    Answer

    The JVM expects main to be void. If you need to return a status code to the OS, use System.exit(code), where code is an int (0 = success by convention).

  • 5. What are args? How do I read command-line input?
    Answer

    String[] args collects command-line tokens. Example:

    public static void main(String[] args) {
    System.out.println("You passed " + args.length + " args.");
    for (String s : args) System.out.println(s);
}

    Run as java MyClass hello 123 → args[0] = "hello", args[1] = "123".

  • 6. Can I use String... args (varargs) instead of String[] args?
    Answer

    Yes. public static void main(String... args) is valid because varargs compile to a String[] at runtime. Under the hood the JVM still sees a String[].

  • 7. Does the name args matter?
    Answer

    No — the parameter name is arbitrary. public static void main(String[] commandLine) works the same. The important part is the type: String[] (or varargs).

  • 8. Simple “Hello world” main example?
    Answer 
    public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}
  • 9. What happens if main is not public?
    Answer

    You can compile a non-public main, but the JVM will not find a public entry point and will throw NoSuchMethodError or similar when you try java ClassName. So to be runnable by the launcher, main must be public.

  • 10. Can I overload main?
    Answer

    Yes — you may define public static void main(String[] args) and public static void main(). But the JVM only calls the exact main(String[]) signature. Overloads are useful for helper entry points you call from code.

  • 11. What is the “main thread”?
    Answer

    The JVM starts a thread named main and runs the main method on it. If you spawn more threads from main, those are additional threads.

  • 12. What happens if an exception is thrown and uncaught in main?
    Answer

    The program prints a stack trace and usually terminates with a non-zero exit. To handle error conditions gracefully, catch exceptions or use throws in the signature to let them bubble up and be visible.

  • 13. How do I parse numbers from args?
    Answer 
    public static void main(String[] args) {
  if (args.length > 0) {
    int n = Integer.parseInt(args[0]); // may throw NumberFormatException
    System.out.println("n = " + n);
  }
}
  • 14. Can I change args inside main?
    Answer

    Yes — args is a normal array reference. You can modify its elements (it’s not final unless you declare it that way). But modifying them only affects the runtime of that JVM process.

  • 15. Can I run a Java program without a main method?
    Answer

    For standard standalone Java apps launched by java, you need a main entry point. Some frameworks (servlets, app servers, Android) use different lifecycle entry points provided by the framework, not a main method.

  • 1. What happens before main runs (class initialization)?
    Answer

    The class’s static initializers and static field initializations run before main. Order: static field initializers and static { ... } blocks execute when the class is initialized — which happens before main is invoked.

    Example:

    class A {
    static { System.out.println("A static init"); }
    public static void main(String[] args) { System.out.println("A main"); }
    }
    // Output:
    // A static init
    // A main
  • 2. Can I declare main with throws Exception?
    Answer

    Yes — public static void main(String[] args) throws Exception is valid. If an exception propagates out, the JVM prints it and exits.

  • 3. How do I call the main method of another class from my main?
    Answer

    Call it like any static method:

    OtherClass.main(new String[] { "hello" });
  • 4. Can main be inside an interface or enum?
    Answer

    Yes. Since Java 8 allows static methods in interfaces, you can put a public static void main in an interface or enum and run java YourInterface or java YourEnum as long as the method is public static void main(String[]).

  • 5. How do I safely start threads from main and wait for them?
    Answer

    t.start();
    t.join(); // wait until t finishes

  • 7. How do I create an executable JAR? What does manifest Main-Class do?
    Answer

    Add Main-Class: com.example.Main to META-INF/MANIFEST.MF. When java -jar app.jar runs, the JVM reads Main-Class and invokes that class’s public static void main(String[]).

  • 8. What’s a common mistake with main and args when using reflection?
    Answer

    When invoking main via reflection, you must pass the String[] as a single Object argument or it will be treated as varargs. Correct:

    m.invoke(null, (Object) new String[] {"a","b"}); // cast to Object

    else if ("-f".equals(args[i])) filename = args[++i];
    }

    For complex cases, use a CLI library (Apache Commons CLI, picocli).

  • 10. Are there signature variations that still run?
    Answer

    The JVM requires a method with descriptor ([Ljava/lang/String;)V. public static void main(String[] args) and public static void main(String... args) both produce that descriptor and run. Different return type (int) or wrong parameter types will not be used by the launcher.

  • 11. What happens with System.exit() inside main?
    Answer

    System.exit(status) terminates the JVM with that exit code. Shutdown hooks run first (unless forcibly halted). Use System.exit to signal success/failure to OS.

  • 12. Can I debug main right from the first line?
    Answer

    Yes, but you typically start the JVM with debugger options (e.g., -agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=*:5005) so the debugger can attach before main begins.

  • 13. Does main support Unicode or spaces in command-line args?
    Answer

    Yes — how the shell passes arguments depends on the operating system and the shell quoting rules. Java receives them as String elements.

  • 14. Can main be used as a quick test harness? Any caveats?
    Answer

    Yes — main is convenient for quick manual tests. Caveat: unit tests (JUnit) are better for repeatability and automation.

  • 15. Is main executed when you run a class through an IDE “Run” action?
    Answer

    Yes — IDEs configure a launcher that calls the JVM and starts main with given arguments and working-directory environment. The IDE may also set classpath, VM options, and environment variables.

    • Hard Question

  • 1. What is the JVM method descriptor for main?
    Answer

    The bytecode descriptor is ([Ljava/lang/String;)V. That means a single parameter of type String[] ([L...;) and V for void.

  • 2. Why does m.invoke(null, (Object) new String[]{"a"}) need that cast?
    Answer

    Reflection invoke uses varargs; without the cast the String[] is expanded as multiple varargs parameters instead of one Object parameter. Casting to Object prevents varargs expansion and passes the array as the single required argument.

  • 3. Can main exist in multiple classes inside the same JAR? How does JVM choose which to run?
    Answer

    You can have many classes with main. If you run java -cp app.jar com.example.Main, JVM runs that main. If you run java -jar app.jar, JVM reads the Main-Class entry in META-INF/MANIFEST.MF to decide which class to invoke. If no Main-Class exists, java -jar fails.

  • 4. How do Java Modules affect launching main?
    Answer

    With modules you can run a module’s main using the -m option:

    java --module-path mods -m my.module/com.example.Main

    The launcher then loads com.example.Main from module my.module and calls its main. The module system controls visibility and packaging at runtime.

  • 5. Can the main method be invoked by a custom class loader? Any issues?
    Answer

    Yes — custom class loaders can load classes and let you invoke their main. Beware of class identity: classes loaded by different class loaders are considered distinct, and static state is isolated per loader. Also ensure the loader has access to core classes.

  • 6. Who runs main — the JVM launcher thread or something else?
    Answer

    The JVM creates an initial thread (usually named main) that executes class initialization and then calls the main method. You can change thread behavior afterwards but the entry thread is the one that calls main.

  • 7. Can you override main via inheritance?
    Answer

    You can hide static methods but not override them. If Sub extends Super and both declare public static void main, they are separate methods. The JVM chooses the main of whichever class you request to run.

  • 8. Is it possible to run a private main using reflection?
    Answer

    Yes — use getDeclaredMethod("main", String[].class), then setAccessible(true), then invoke. But the standard launcher will still not run a non-public main.

  • 9. How do class initializers relate to memory visibility and the Java Memory Model?
    Answer

    Class initialization establishes a happens-before relationship: once a class is initialized, the values assigned during static init are visible to threads that see the initialized class. The thread calling main will see fully-initialized static fields.

  • 10. Can main be used to start a framework like Spring Boot? What is typical?
    Answer

    Yes — typical pattern:

    public static void main(String[] args) {
    SpringApplication.run(MyApp.class, args);
    }

    Main acts as a bootstrapper: it gathers args, configures logging, and starts the application’s lifecycle.

  • 11. What are pitfalls with using static state initialized before main?
    Answer

    If static initializers do heavy I/O or rely on environment that isn’t ready, you can slow or break startup. Keep static initialization minimal; prefer lazy initialization if possible.

  • 12. How do you debug very early startup (before main)?
    Answer

    Start JVM with debug options and suspend=y so the debugger attaches before the VM proceeds to class initialization and main. Example:

    java -agentlib:jdwp,transport=dt_socket,server=y,suspend=y,address=*:5005 -jar app.jar
  • 13. Can a non-void main work if you launch via an application launcher?
    Answer

    No — the JVM requires void for the entry point. A different return type like int will not be used by the launcher. Use System.exit(int) to communicate a numeric status.

  • 14. How does main interact with shutdown hooks?
    Answer

    If main calls System.exit, registered shutdown hooks run before the JVM exits (unless forcibly terminated). If main simply returns and no non-daemon threads exist, the JVM exits and hooks run too.

  • 15. Advanced: What about calling main in embedded or agent scenarios (jvmti, instrumentation)?
    Answer

    Tools and agents can instrument or load classes before main runs (via premain / agentmain). Agents can modify bytecode or set up environment so main runs under instrumentation. This is how profilers, APMs, or test coverage tools hook into application startup.
